## PHBServer (phbcli + phbgateway)

Install [uv](https://docs.astral.sh/uv/getting-started/installation/) — the Python package and project manager used by the server.

### First-time setup

After cloning, run from the repo root:

```bash
cd phbserver
uv sync
uv tool install --editable phbcli
```

`uv sync` installs all workspace dependencies (shared by both `phbcli` and `phbgateway`). `uv tool install --editable phbcli` installs both the `phbcli` and `phb-channel-devices` binaries — the mandatory channel is declared as a script in `phbcli`'s `pyproject.toml`, so a single tool install covers both.

`phbgateway` does not need a separate tool install — it runs directly from the workspace via `uv run phbgateway`.

### After pulling updated code

Use the helper script at the repo root — run this every time you pull changes or switch environments:

```bash
./dev-sync.sh
```

It runs the following steps:

```bash
phbcli stop          # release Windows file lock on the binary before overwriting
cd phbserver
uv sync
uv tool install --editable phbcli --upgrade --force
```

<Warning>
`uv sync` alone is **not enough** after code changes. `phbcli` is a `uv tool` — its entry-point scripts are baked at install time. If `pyproject.toml` or the package structure changes, the old script will point at a stale or missing module and fail with `ModuleNotFoundError`. Always run `./dev-sync.sh` after pulling. On Windows the server must be stopped first so the binary file lock is released.
</Warning>

## Mobile / web app

Install [Flutter](https://docs.flutter.dev/install/quick).

Then install [FVM](https://fvm.app) (Flutter Version Manager) to use the exact Flutter version pinned in the repo:

```bash
dart pub global activate fvm
```

On **Windows (Git Bash)**, add the following to `~/.bashrc`:

```bash
export PATH="$PATH:/c/Users/<YOU>/AppData/Local/Pub/Cache/bin"
alias fvm='fvm.bat'
```

On **Windows**, also enable **Developer Mode** (Settings → System → For developers) so FVM can create the symlinks it needs.

Once inside the `device_apps` folder, install the pinned Flutter version:

```bash
fvm install
```

Use `fvm flutter` in place of `flutter` for all commands, e.g.:

```bash
fvm flutter build apk --split-per-abi
fvm flutter run
```

## Diagram generator

The diagram generator scans `mintdocs/` for mermaid code blocks, extracts them to `diagram-gen/sources/`, and renders each to a PNG in `mintdocs/images/diagrams/`.

Install Node.js, then install the Mermaid CLI globally:

```bash
npm install -g @mermaid-js/mermaid-cli
mmdc --version
```

Run from the repo root to render only changed or new diagrams:

```bash
uv run python diagram-gen/generate.py
```

To re-render all diagrams regardless of changes:

```bash
uv run python diagram-gen/generate.py --force
```

See [Diagram generator](/build/diagram-generator) for full authoring and workflow details.
